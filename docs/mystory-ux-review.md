# 나의이야기 — UI/UX 점검 및 개선 계획

> 점검 기준: 65세+ 어르신 직접 사용 + 인터뷰어(가족/도우미) 보조 사용
> 현재 코드: mystory-app.jsx (831줄, React 프로토타입)

---

## 1. 사용 시나리오별 점검

### 시나리오 A: 어르신이 혼자 사용 (스마트폰/태블릿)

| 화면 | 현재 상태 | 문제점 | 심각도 |
|------|----------|--------|--------|
| 랜딩 | 제목/저자 입력 → 시작 | "BOOK TITLE", "AUTHOR" 영문 라벨이 어르신에게 생소 | 🟡 |
| 주제 선택 | 72개 카드, 아코디언 | 카드가 너무 많아 선택에 부담. 추천/가이드 없음 | 🔴 |
| 목차 편집 | ↑↓× 버튼 | 작은 32px 아이콘, 드래그 미지원, 목적이 불분명 | 🟡 |
| 대화 모드 | AI 질문 → 답변 | 진행 상황(몇 번째 질문인지) 알 수 없음 | 🔴 |
| 대화 모드 | "다음 장 →" 버튼 | 현재 장이 끝났는지, 더 이야기해도 되는지 안내 없음 | 🔴 |
| 일반 모드 | 빈 textarea | 가이드 닫으면 막막. 시작 문구 도움 없음 | 🟡 |
| 사이드바 | TEXT SIZE, VOICE INPUT | 영문 라벨, 설정이 숨겨져 있어 발견 어려움 | 🟡 |
| 책 미리보기 | 표지 + 본문 | "← 편집" 텍스트 버튼이 작고 불분명 | 🟡 |

### 시나리오 B: 인터뷰어가 보조하며 사용

| 화면 | 현재 상태 | 문제점 | 심각도 |
|------|----------|--------|--------|
| 전체 | 인터뷰어용 가이드 없음 | 인터뷰어가 어떻게 진행해야 하는지 안내 부재 | 🔴 |
| 대화 모드 | AI 자동 질문만 존재 | 인터뷰어가 직접 질문을 수정/추가할 수 없음 | 🔴 |
| 전체 | 역할 구분 없음 | 인터뷰어/어르신 모드 구분이 없어 설정 노출 | 🟡 |
| 전체 | 세션 관리 없음 | 중간에 끊었다가 다시 이어하기 불가 | 🔴 |
| 책 미리보기 | PDF 내보내기 없음 | 결과물 공유/인쇄 불가 | 🟡 |

## 2. 화면별 상세 개선 사항

### 2-1. 랜딩 페이지

**현재**: 제목/저자 입력 → 시작하기

**개선**:
- 영문 라벨("BOOK TITLE", "AUTHOR") → 한글로 변경 ("책 제목", "지은이")
- 간단한 서비스 소개 일러스트 또는 완성된 책 예시 이미지 추가
- "어떻게 만들어지나요?" 간단 3단계 안내 (주제 선택 → 대화 → 책 완성)
- 이전에 작성 중이던 책이 있으면 "이어서 작성하기" 버튼 노출 (데이터 저장 선행)

### 2-2. 주제 선택 페이지

**현재**: 10개 카테고리, 72개 카드 전체 나열

**개선**:
- **추천 프리셋 추가**: "처음이세요? 이 5개로 시작해보세요" 원터치 선택
  - 예: 탄생 이야기, 가장 오래된 기억, 배우자와의 만남, 가장 뿌듯한 성과, 인생에서 배운 것
- **단계별 안내**: "많이 고를수록 풍성한 책이 됩니다. 최소 3개를 권장합니다"
- 카테고리 기본 닫힘은 유지하되, **첫 번째 카테고리는 자동 펼침** (진입 시 빈 화면 방지)
- 선택 카운터를 상단에 눈에 띄게 표시: "현재 N개 선택됨"

### 2-3. 대화 모드 (핵심 개선 필요)

**현재**: AI가 하드코딩 질문 → 사용자 답변 → 반응+다음 질문 반복

**개선 — 대화 흐름**:
- **진행률 표시**: 상단에 "질문 2/5" 또는 프로그레스 바
- **"더 이야기하기" 버튼**: 하나의 질문에 여러 번 답변 가능하도록. 현재는 답변 하나에 바로 다음 질문으로 넘어감
- **"이 주제 마치기" 버튼**: 사용자가 원할 때 다음 챕터로 넘어가기 (현재 "다음 장 →"의 역할이 모호)
- **맥락 기반 후속 질문**: "좀 더 자세히 말씀해주세요", "그때 기분이 어떠셨어요?" 같은 깊이 있는 후속 질문 (AI 연동 시 해결)

**개선 — 입력 UX**:
- **음성 입력 시 자동 전송 옵션**: 녹음 종료 후 바로 전송 (어르신이 전송 버튼 따로 누르기 어려움)
- 입력창 placeholder를 맥락에 맞게 변경: "태어나신 곳과 년도를 말씀해주세요..." (현재 항상 동일)

**개선 — 시각적 피드백**:
- AI 말풍선에 타이핑 완료 후 부드러운 등장 애니메이션
- 사용자 메시지 전송 후 즉각적 시각 피드백 (체크마크 등)

### 2-4. 일반 모드 (글쓰기)

**현재**: 빈 textarea + "가이드" 버튼

**개선**:
- **시작 도우미**: "이 주제에 대해 이렇게 시작해보세요:" + 예시 첫 문장 제안
  - 예: "나는 1953년 봄, ___에서 태어났다."
- **글쓰기 힌트**: 우측 하단에 떠있는 작은 "?" 버튼 → 탭하면 질문 힌트 목록 표시
- **글자 수 외에 예상 페이지 수** 표시: "약 2페이지 분량"
- **자동 저장 인디케이터**: "저장됨" 표시 (데이터 저장 구현 후)

### 2-5. 사이드바

**현재**: CONTENTS, TEXT SIZE, VOICE INPUT, 책 미리보기

**개선**:
- 영문 라벨 한글화: "목차", "글자 크기", "음성 입력"
- **전체 진행률 표시**: "7개 중 3개 작성 완료 (43%)" + 프로그레스 바
- 완료/미완료 챕터 시각 구분 강화 (현재 ✓만 표시 → 배경색 차이도 추가)
- **"전체 저장" 버튼** 추가 (데이터 영속성 구현 후)

### 2-6. 책 미리보기

**현재**: 표지 + 목차 + 본문 (스크롤)

**개선**:
- **챕터별 이동**: 목차 항목 클릭 시 해당 챕터로 스크롤
- **PDF 다운로드 버튼**: "책 저장하기" (서버사이드 PDF 생성)
- **공유 버튼**: 링크 생성하여 가족에게 공유
- **인쇄용 레이아웃 토글**: 웹 보기 ↔ 인쇄용 A4
- 표지 디자인 선택 (템플릿 2~3종)

## 3. 인터뷰어 모드 (신규 기능)

실제 서비스에서는 **인터뷰어(가족/도우미/직원)**가 어르신 곁에서 도와주는 경우가 대부분입니다.

### 3-1. 인터뷰어 가이드 화면

```
인터뷰 시작 전 안내 (인터뷰어에게만 보임):
┌──────────────────────────────────────────┐
│  이 챕터의 인터뷰 가이드                    │
│                                           │
│  주제: "나의 탄생 이야기"                   │
│                                           │
│  핵심 질문:                                │
│  1. 태어난 시기와 장소                      │
│  2. 출생 당시 가족 상황                     │
│  3. 이름의 유래                             │
│                                           │
│  팁: 구체적인 장소나 날짜를 여쭤보면         │
│  기억이 더 선명해지십니다.                   │
│                                           │
│  [인터뷰 시작하기]                          │
└──────────────────────────────────────────┘
```

### 3-2. 인터뷰어 도구

- **질문 커스터마이즈**: AI 자동 질문 외에 인터뷰어가 직접 질문 입력
- **메모 기능**: 인터뷰어가 어르신 반응이나 비언어적 표현을 메모 (책에는 미포함)
- **일시정지/재개**: "잠시 쉬어가기" 버튼 (어르신 피로도 관리)
- **세션 타이머**: 인터뷰 경과 시간 표시 (30분 넘으면 알림)

### 3-3. 역할 전환

```
앱 진입 시:
┌─────────────────────────┐
│  나의이야기               │
│                          │
│  [혼자 작성하기]          │  → 기존 전체 흐름
│  [도움 받아 작성하기]     │  → 인터뷰어 모드 (설정 관리는 인터뷰어)
│  [관리자]                │  → 관리자 대시보드 (추후)
└─────────────────────────┘
```

## 4. 접근성 개선 (어르신 필수)

| 항목 | 현재 | 개선 |
|------|------|------|
| 색상 대비 | mute(#A8A29E) 텍스트 대비 부족 | WCAG AA 기준 4.5:1 이상으로 조정 |
| 터치 타겟 | 대부분 44px+ 확보 | 목차 편집 SmBtn(32px)은 미달 → 44px로 |
| 포커스 표시 | outline:none 적용 | 키보드 사용 시 포커스 링 표시 |
| 스크린리더 | ARIA 라벨 없음 | 버튼, 입력란에 aria-label 추가 |
| 모션 감소 | 애니메이션 항상 활성 | prefers-reduced-motion 대응 |
| 에러 안내 | 없음 | 음성인식 실패 시 "다시 시도해주세요" 등 |
| 텍스트 크기 | 일반/확대 2단계 | 특대(24px+) 추가 검토 |

## 6. 핵심 질문 풀 보강 필요

현재 질문 풀 상태:

| 주제 ID | 전용 질문 수 | 비고 |
|---------|------------|------|
| birth-story | 3 | ✅ 충분 |
| parents-story | 3 | ✅ |
| first-memory | 3 | ✅ |
| childhood-home | 2 | 🟡 1~2개 추가 권장 |
| best-friend | 2 | 🟡 |
| spouse-meeting | 2 | 🟡 |
| life-wisdom | 2 | 🟡 |
| epitaph | 2 | ✅ |
| 나머지 64개 주제 | 공통 4개만 | 🔴 전용 질문 없음 |

→ 72개 주제 중 **64개가 공통 질문(`_`)만 사용**. 주제별 맞춤 질문이 없으면 "나의 취미"든 "군대 이야기"든 같은 질문("이 시절 가장 먼저 떠오르는 기억이 있나요?")이 나옴. **AI 연동 전까지 최소 상위 20개 주제에 전용 질문 3개씩 추가 필요.**

## 7. 현재 코드에서 즉시 보완 가능한 기술 이슈

| 이슈 | 위치 | 설명 |
|------|------|------|
| 데이터 휘발 | App state | 새로고침 시 모든 작성 내용 소실 → localStorage 즉시 적용 권장 |
| 챕터 전환 시 ChatEd 초기화 | ChatEd useEffect | 다른 챕터 갔다 돌아오면 qI가 리셋되어 같은 질문 재시작 |
| textarea 높이 고정 | NormalEd | 텍스트가 많아져도 minHeight:350 고정 → auto-grow 필요 |
| 사진 순서 | NormalEd photos | 사진 순서 변경 불가, 텍스트 중간 삽입 불가 |
| 책 미리보기 재진입 | BookPage | 편집으로 돌아갈 때 마지막 편집하던 챕터가 아닌 0번으로 이동 |

## 요약

현재 프로토타입은 **전체 흐름은 완성**되어 있으나, 어르신이 혼자 쉽게 사용하기에는 **안내/가이드 부족**, 인터뷰어가 보조하기에는 **역할 구분/도구 부재**가 가장 큰 갭입니다.

가장 임팩트 있는 개선 3가지:
1. **영문 라벨 한글화 + 대화 진행률 표시** — 즉시 적용 가능, 체감 효과 큼
2. **데이터 저장 (localStorage)** — 어르신이 실수로 앱을 닫아도 내용 유지
3. **인터뷰어 모드 기본 구현** — 실서비스 핵심 시나리오
