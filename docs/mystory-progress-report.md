---

## 1. 프로젝트 개요

어르신 대상 AI 기반 자서전 제작 플랫폼.
대화형 인터뷰(챗봇) 또는 직접 글쓰기로 인생 이야기를 챕터별로 작성하고,
한 권의 책으로 완성하는 웹 서비스.

### H. 이번 세션 버그 수정

| 버그 | 원인 | 수정 |
|------|------|------|
| 대화 시 사용자 답변이 사라짐 | WritePage의 `upd` 함수가 stale closure | ChatEd가 `setChs` 직접 사용 + `idxRef`로 최신 인덱스 유지 |
| 같은 질문 두 번 반복 | `qI.current` 증가 타이밍 문제 | `send()` 호출 즉시 인덱스 확정 후 증가 |
| 사진 버튼이 더미 이미지 삽입 | 데모용 하드코딩 | `<input type="file">` + FileReader로 실제 파일 선택 |
| "느티나무" 데모 텍스트 자동 삽입 | 녹음 시뮬레이션 로직 | 시뮬레이션 코드 제거, 실제 STT 연결 |

## 6. 권장 다음 단계

**가장 임팩트 있는 다음 작업 3가지:**

1. **Next.js 프로젝트 세팅 + 프로토타입 통합** — `mystory-app.jsx`를 실제 Next.js 프로젝트의 페이지 컴포넌트로 분리. 이것이 모든 후속 작업의 기반이 됩니다.

2. **AI 대화 실연동** — 현재 하드코딩된 `REACTIONS`와 `getQ()`를 `useBiographer` 훅 + `/api/ai/biographer/next-message` API로 교체. 이것만 되면 실제 데모 가능.

3. **데이터 저장** — 작성 중 데이터를 localStorage(단기) 또는 DB(장기)에 저장. 새로고침 시 데이터 유실 방지. 어르신 사용자에게 치명적인 UX 문제.
